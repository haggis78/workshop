Uploading your data

  1. Go to https://kumu.io/. If you did not yet create an account, click the blue "JOIN FOR FREE" button and do so now.
  2. Click the green "New Project" button at upper right.
  3. Give the project a name. Description is optional. Tick "I'm not a robot" and then "Create project".
  4. Optionally, give your map a name.
  5. Pick the "Systems" template.
  6. Our graph is directional, so choose "Connections will be => Directed", then "Get started".
  7. You now have a blank graph. To upload data, click the green plus button at bottom center, and choose "Import".
  8. Start with your nodes table. You can "Choose file" and upload submission-nodes.csv from your computer, or paste in the link to a Google Sheet if that's where your data are. Then click "Import spreadsheet".
  9. You now see your data in "Import review". By naming our first two columns "Label" and "Type", we have already told Kumu what those columns represent; if you used different column labels, you would need to use dropdowns to select (at minimum) what column to use for Label. Click "Save import". You will now see all of your nodes.
  10. Now do the same with your edges table. You can "Choose file" and upload submission-edges.csv from your computer, or paste in the link to a Google Sheet if that's where your data are. Then click "Import spreadsheet".
  11. You will again see your data in "Import review". By naming our first two columngs "From" and "To", we have already told Kumu what those columns represent; if you used different column labels, you would need to use dropdowns to select what column to use for each, even if your graph is non-directional or bi-directional. Click "Save import". You will now see all of your edges.

Styling your network graph

  1. Click the slider icon at the upper right. This opens the control panel.
  2. Start with "Size by". Select "frequency". This will size the nodes according to how often someone appears.
  3. Next, click "Color by". Select "Element Type" to select that column from our data. This will turn the Witness nodes one color and the Submitter nodes another. You can also change the color scheme.
  4. Now let's spread out the graph so we can see better. Click "Customize defaults", go down to Layout Settings, and change Preset from auto to hairball.
  5. Try dragging nodes around; you'll see them pull on one another. To pin them in place, so you can drag them one at a time, go down to "Layout", and change from force-directed to "off".
  6. Once you like what you see, click "Save". If you want to start your styling over, click "Revert", and it will go back to the last saved version.

Sharing your network graph

Once you have your graph in the form you want to share, click the project name in the upper left corner. Select "EMBEDS < / > " . Give it a title. Optionally, select "Include map overview in sidebar" and/or "Include current position and scale", then "Create embed / share link". You'll get both a read-only link and an html embed code. If using this as a classroom assignment, students can submit this to you for you to view and grade.

Tagging text in Recogito

Preparing to tag

  1. Go to https://recogito.pelagios.org/. If you did not do so earlier, click the blue "Register for Recogito" button and create an account now. If you have an account, click "Log In" at the top right.
  2. If you did not do so already, click here to download the cleamed text of Eirik's Saga.
  3. In Recogito, create a new project by clicking the + New button in the upper left. As we are working with just one file, select "File upload". Navigate to your Download folder to locate the file Eiriks-saga-text-cleaned.txt. Click "Open" to upload it to Recogito. In a moment, you will have it listed on Recogito as a document; click to open.
  4. At the top, make sure Annotation Mode is selected as Normal, and change Colour to By Entity Type.

Tagging Persons

  1. Highlight "Olaf the White". Select "Person". Click "Add tag..." and type in "King Olaf the White Ingjaldson". Click "OK" to save.
  2. Similarly, tag "King Ingjald Helgison".
  3. When you try to tag "Helgi" as "Helgi Olafson", it will ask if you want to tag all instances of "Helgi" in the text the same way. Select no: there's a different Helgi somewhere else.
  4. Tag the rest of the people in the sentence.
  5. If any of the highlights are yellow rather than blue, you forgot to select "Person". Go back and rectify that.

Tagging and Viewing Places

  1. Highlight "Dublin" and select "Place". Recogito should automatically match Dublin in its internal gazetteers. Click "Confirm" and then "OK".
  2. Do the same with "Ireland" and "Hebrides".
  3. At the top, click the "Map View" icon. You'll see a dot on Dublin, but not on Ireland or the Hebrides. Why?
  4. Click the Dublin dot to see the relevant text.
  5. Note: It is also possible to tag Events, but we won't be using that today.

Tagging Relationships

  1. Return to "Document View" (top left, square with pencil icon). Scroll back down to your tagged persons.
  2. Under "Annotation Mode" in the top menu, select "Relations".
  3. Click on King Ingjald, then on Olaf the White. In the "Tag..." space, type "father of" and click the checkmark.
  4. Similarly tag the relations among the other people whose names you've tagged.
  5. Note: It is possible to tag relationships among any type of entity: persons, places, events. Our task today deals only with relations among persons.

Processing Recogito XML Download through XQuery (demonstration only)

XQuery is a query language that allows the user to treat an XML document like a database. It can be written to locate whatever information you want in the XML and create an output file in whatever format you choose--but it does require understanding both XML and the output format. It also requires use of either eXist-DB or oXygen XML Editor software. eXist-DB is free, but I am not current on its use. oXygen is proprietary and expensive, but students can download and install it with a free 30-day trial license.

The XQuery files I created for this project are structurally defined and content-agnostic. This means they should work whether your input document is an Icelandic saga or a telephone book. Students don't have to write a single line of code, or even understand how it works. All they have to do is follow directions and hit enter to generate the CSV files for Kumu based on their Recogito markup. I also wrote XQuery files for relations between people and places, and between people and events. If you are interested in trying out this activity, let me know and I'll be happy to offer further explanation, demonstration, and support.
